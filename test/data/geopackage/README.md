# GeoPackage Test Fixtures

## example_v2_2.gpkg

Minimal synthetic v2.2 hydrofabric GeoPackage (3 catchments, 2 nexuses).
Generated by `make_v2_2_fixture.py`.

### Topology

```
cat-1 (wb-1) ─┐
               ├─> nex-1 ─> wb-2 (cat-2) ─> nex-2 ─> coastal-000001
cat-3 (wb-3) ─┘
```

### Feature IDs

| Layer     | ID       | toid            | Notes                          |
|-----------|----------|-----------------|--------------------------------|
| divides   | cat-1    | nex-1           | divide_id; id=wb-1             |
| divides   | cat-2    | nex-2           | divide_id; id=wb-2 (mainstem)  |
| divides   | cat-3    | nex-1           | divide_id; id=wb-3             |
| nexus     | nex-1    | wb-2            | confluence of cat-1 and cat-3  |
| nexus     | nex-2    | coastal-000001  | terminal nexus                 |
| flowpaths | wb-1     | nex-1           | within cat-1                   |
| flowpaths | wb-2     | nex-2           | within cat-2 (mainstem)        |
| flowpaths | wb-3     | nex-1           | within cat-3                   |

### Schema (v2.2)

- `nexus`: id, toid, type, vpuid, poi_id
- `divides`: divide_id, id, toid, type, ds_id, areasqkm, vpuid, lengthkm, tot_drainage_areasqkm, has_flowline
- `flowpaths`: id, toid, mainstem, hydroseq, lengthkm, areasqkm, tot_drainage_areasqkm, has_divide, divide_id, poi_id, vpuid

`detect_version` identifies this file as V2_2 because `nexus.id` is present (not `nexus.nexus_id`).

## Regenerating fixtures

The committed `.gpkg` files for `example_v2_2`, `example_v3_0`,
`example_v3_0_minimal`, `example_v3_0_dangling`, and `example_v3_0_extra_col`
are byte-deterministic. Re-running any `make_*.py` against an unchanged
generator produces an identical file (verifiable with `md5`).

This relies on `gpkg_contents.last_change` being pinned to the literal
string `'2026-01-01 00:00:00'` in every generator (rather than
`datetime('now')`). Do not revert that pin: every regeneration would then
produce a spurious working-tree diff even when nothing meaningful changed.
